Описание экзаменатора UTISE
Работа программы в обычном режиме. При запуске программы выводится окно экзаменатора UTISE (рис. 21).
После щелчка мышью на кнопке Открыть в выведеном списке файлов можно выбрать любой тестовый файл с расширением .tfs, при запуске которого нужно ответить на все вопросы-задания, хранящиеся в нем, а при выборе файла с расширением .tfp – на заранее установленное число заданий, выбираемых случайным образом из базы тестовых заданий по соответствующей теме курса.
Рис. 21
Выбор заданий из проекта теста происходит следующим образом: из каждого файла, входящего в тест, выбирается одинаковое число заданий. Например, если в тест включено два файла и требуется выбрать десять заданий, то из каждого файла будет выбрано случайным образом по пять заданий.
Работа программы с командной строкой. Программе с командной строкой можно передать следующие параметры:
1. Имя тестового файла или файлов проекта с указанием пути к них.
2. Путь к реестру, куда программно должен быть записан результат тестирования (см. 6.5, предоставление результата другим приложениям). З а м е ч а н и е. Эта процедура в этой версии не реализована!?.
По завершению сессии тестирования выход из программы осуществляется автоматически.
Настройки проекта (теста) по умолчанию. При загрузке тестового файла с расширением .tfs настройки проекта принимают значения по умолчанию, а именно:
Число заданий – все задания, хранящиеся в тестовом файле;
Время сессии тестирования – нет (не ограничено);
Возврат (к пропущенному тестовому заданию) – да (разрешено);
Метка (возможность пометить задания для повторного их анализа и выбора ответов до завершения сессии тестирования) – да;
Подсчёт (баллов) – разносный (сдал, не сдал);
Стиль отметки (результата тестирования) – дифференцированный (отлично, хорошо, удовл., неудовл.);
Отметка: отлично – ≥ 90%; хорошо – ≥ 75%; удовлетворительно – ≥ 60%; неудовлетворительно – < 60% от всех правильных ответов – эталонов заданий);
Отчет – графический, в том числе, по каждому запрашиваемому заданию.
Переход к предыдущему вопросу осуществляется нажатием кнопки . Если параметр проекта Возврат не установлен, то перейти к предыдущему вопросу нельзя. Переход к следующему вопросу осуществляется нажатием кнопки .
Строка состояния. Внизу окна UTISE-экзаменатора расположена строка состояния. Она разделена на три части:
1. Краткая справка, появляющаяся при удержании курсора мыши над какой-нибудь кнопкой.
2. Номер текущего задания / общее число заданий.
3. – время, оставшееся до окончания сессии тестирования.
Подсчет и вывод результатов тестирования. Результаты тестирования выводятся после щелчка мышью на кнопке Закончить.
Если установлен параметр проекта Возврат, то перед выводом результата тестируемый может заменить или оставить без изменения выбранные ответы помеченных заданий (рис. 22)
Рис. 22
Пометки напротив вопросов (заданий):
– в данном задании не был отмечен ни один ответ;
– вопрос был помечен (см. 6.4, создание проекта).
Чтобы перейти к выбранному вопросу нужно щёлкнуть мышью на кнопке Перейти.
Вывод результата тестирования осуществляется посредством щелчка мышью на кнопке 'Закончить'. В зависимости от параметра проекта Отчет, результат выводится в cтатистическом (рис. 23) или графическом виде (рис. 24).
В верхней части окна указывается тема проекта. В рамке указан результат.
Статистический результат:
Максимально возможное число существенных операций;
Минимальное число правильно выполненных существенных операций, требуемое для успешной сдачи теста.
Ваш результат – число существенных операций, набранное тестируемым.
Рис. 23
Результат, выраженный в графической форме:
Синяя полоска – процент от числа существенных операций, при достижении которого тест считается сданным;
Зеленая полоска – тест сдан;
Красная полоска – тест не сдан.
Рис. 24
При нажатии кнопки 'Подробно', выводится статистика по каждому вопросу и темам, если установлены соответствующие параметры проекта (рис. 25).
